# 安卓脚手架开发规范(Java-MVC)

### 布局
+ 控件命名格式为:控件缩写_(位置)_功能，具体命名规范见工作室wiki
+ 字符串、颜色全部放在资源文件中
+ 相同样式统一定义在style中
+ 避免过多的布局嵌套，尽量使用相对布局
+ 引入的图片单张不能超过500k，尽量减少打包体积

### Java
+ 包名全小写，类名大驼峰，方法、变量名小驼峰。
+ 常量全大写配合下划线
+ 静态变量加s前缀，成员变量加m前缀(AS自动配置)
+ 所有活动根据需要统一继承`BaseActivity`或`BaseToolActivity`
+ 避免可能出现的内存泄漏，用LeakCanary进行内存检查

### 其他
+ 慎引第三方库，否则可能导致打包失败
